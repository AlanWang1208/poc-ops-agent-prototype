独立 Compare 工作区

后续适合做批量对账、历史报告和大表任务。第一版投入较大，可能打断当前 SQL 工作台节奏。

推荐

  • 第一版选 A：把 Compare 当成 SQL 工作台里的一个模式。
  • 原因：复用连接上下文、执行状态、AI 分析和结果面板，不引入新的工作区系统。

暂缓

  • 暂不做历史报告、批量任务、字段映射和 N 数据源矩阵。
  • 暂不支持生产连接，仍限定 development/test。

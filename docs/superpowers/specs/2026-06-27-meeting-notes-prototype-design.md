# 会议录制纪要前端原型设计

- 状态：已确认设计范围
- 日期：2026-06-27
- 相关模块：M01、M02、M04、M09、M10、M11
- 实施类型：前端原型，不接真实后端

## 1. 目标

本轮将 `/meeting-notes` 从受保护占位页升级为会议纪要管理前端原型，用于验证会议纪要维护、条件搜索、会议 RAG 问答和录制入口的信息架构。

第一版以“纪要列表优先”为主体验：

1. 操作员进入页面后先看到会议纪要库。
2. 支持按标题、关键词、参会人、时间范围、项目和标签筛选会议纪要。
3. 支持进入独立详情页查看 AI 摘要、关键决策、行动项、全文转写和版本记录。
4. 支持全局会议知识问答和单场会议限定问答的原型展示。
5. 支持客户端 Python 录制程序配置原型。
6. 支持包含配置检查的四步录制向导原型。
7. 支持草稿校订与发布新版本的原型展示。

本轮不接入真实 Python 录制程序，不请求麦克风权限，不调用真实大模型，不建立真实 RAG 索引，不新增后端契约，不写入后端数据。

## 2. 页面结构

### 2.1 纪要库首页

路径：`/meeting-notes`

首页承载会议纪要库主流程：

- 顶部显示页面标题、当前范围和“开始录制”入口。
- 主区域显示筛选条件和纪要列表。
- 筛选条件包含标题、关键词、参会人、时间范围、项目和标签。
- 每条纪要展示标题、会议时间、参会人、项目标签、摘要片段、行动项数量和版本状态。
- 点击纪要进入独立详情页。
- 页面提供空状态、无结果状态和清除筛选入口。

右侧辅助区展示：

- 全局会议问答入口。
- 最近录制或待校订纪要。
- 录制入口、客户端录制程序配置状态和受控状态说明。

### 2.2 纪要详情页

路径：`/meeting-notes/:id`

详情页第一屏突出可维护的会议结论：

- 会议标题、时间、项目、标签、参会人、录制来源和当前版本。
- AI 摘要。
- 关键决策。
- 行动项，包含负责人、截止时间和状态。
- 单场会议问答，引用仅来自当前会议。

详情页下半部分展示：

- 全文转写，按发言人和时间线分段。
- 附件和来源文件。
- 版本记录。
- 草稿状态和最近发布信息。

RAG 回答中的引用应能定位到对应转写片段。原型阶段可使用本地锚点模拟定位，不接真实检索服务。

### 2.3 客户端录制程序配置

路径：`/meeting-notes/recording-settings`

由于会议录制发生在操作员客户端 PC 上，页面需要提供本机 Python 录制程序配置原型。配置归属采用“每个操作员 PC 独立配置”模式：不同操作员、不同设备可以配置不同的 Python 解释器路径、脚本路径和输出目录。

配置项包括：

- Python 解释器路径，例如 `C:\Python311\python.exe`。
- 会议录制脚本路径，例如 `D:\meeting-recorder\record.py`。
- 工作目录。
- 默认输出目录。
- 默认参数模板，例如音频来源、语言、总结模板和输出格式。
- 程序版本或校验摘要。
- 最近一次配置检查结果。
- 当前客户端设备标识或设备备注。
- 当前操作员账号。

原型提供“检查配置”和“保存配置”按钮，但只更新本地模拟状态，不真实读取文件系统，不启动 Python，不执行命令。原型中的配置可视为当前浏览器会话内的本机偏好，不代表服务端已经保存真实客户端路径。

真实落地时，浏览器页面不得直接启动本机任意程序。需要设计受控客户端桥接能力，例如本机桌面客户端、受控本地 Agent 或明确 allowlist 的本机服务，并满足：

- 只能启动已配置、已校验、在 allowlist 内的录制程序。
- 启动参数必须结构化，不允许拼接任意命令行。
- 必须记录操作员身份、配置版本、参数摘要、启动时间、退出状态和输出文件引用。
- 必须向用户明确展示将启动的本机程序、脚本路径和工作目录。
- 真实执行前必须有服务端策略、审计和运行手册约束。
- 每台客户端的实际路径由该客户端本地保存；服务端只保存必要的设备注册状态、配置摘要、策略结果和审计证据。
- 管理员可以下发允许的程序名称、脚本签名、目录范围和参数模板，但不强制所有 PC 使用同一个绝对路径。

### 2.4 录制向导

路径：`/meeting-notes/record/new`

录制向导采用四步流程：

1. 检查客户端录制程序配置，例如 Python 路径、脚本路径和输出目录。
2. 选择音频来源，例如系统音频、麦克风或上传录音文件。
3. 填写会议元信息，例如标题、项目、标签和参会人。
4. 选择总结模板，例如故障复盘、需求评审、日常站会或通用纪要。

完成向导后进入模拟录制状态页或状态区，展示录制中、转写中、总结生成中和已生成纪要等状态。

原型不得触发真实录音，不得请求浏览器麦克风权限，不得启动本机 Python 程序。真实接入时必须通过受控执行链路、身份、授权、审计和运行手册约束。

### 2.5 草稿校订页

路径：`/meeting-notes/:id/edit`

校订流程采用“草稿再发布”：

- 从详情页进入“编辑草稿”。
- 可模拟编辑 AI 摘要、关键决策、行动项和全文转写。
- “保存草稿”只更新本地草稿状态。
- “发布版本”模拟生成新正式版本。
- 版本记录展示版本号、发布人、发布时间和变更摘要。

真实落地时，发布版本属于变更类能力，必须经过服务端授权、审计、版本记录和回滚路径，不得由前端直接决定权限或绕过控制面。

## 3. RAG 问答体验

原型包含两种问答入口：

1. 全局会议知识问答：位于纪要库首页，问题面向全部会议纪要。
2. 单场会议问答：位于详情页，问题限定在当前会议。

问答结果必须展示引用来源：

- 引用的会议标题。
- 会议时间。
- 片段摘要。
- 详情页内的转写定位。

当无引用命中时，原型应展示“未找到可引用内容”，而不是生成无来源回答。真实落地时，RAG 回答必须遵守“无引用不回答”或同等可信约束，并通过服务端契约和评测验证。

## 4. 数据模型

前端原型使用本地模拟数据，建议模型如下：

- `MeetingNote`
- `MeetingActionItem`
- `MeetingDecision`
- `TranscriptSegment`
- `MeetingVersion`
- `MeetingRagMessage`
- `RagCitation`
- `RecordingWizardState`
- `RecordingProgramSettings`
- `MeetingDraft`

模拟数据不得包含真实会议内容、生产数据、密钥、隐私数据或未脱敏日志。

## 5. 组件拆分

建议组件边界：

- `MeetingNotesPage`：纪要库首页。
- `MeetingNotesFilters`：筛选条件。
- `MeetingNoteCard`：列表卡片。
- `GlobalMeetingRagPanel`：全局会议问答。
- `RecordingSettingsPanel`：客户端录制程序配置状态。
- `MeetingNoteDetailPage`：纪要详情页。
- `MeetingSummaryPanel`：摘要、决策和行动项。
- `TranscriptTimeline`：全文转写时间线。
- `MeetingVersionHistory`：版本记录。
- `MeetingScopedRagPanel`：单场会议问答。
- `RecordingWizardPage`：录制向导。
- `RecordingSettingsPage`：客户端录制程序配置页。
- `MeetingDraftEditorPage`：草稿校订页。

组件内部继续使用 JavaScript/JSX、JSDoc、`checkJs` 和 Zod 边界校验的现有前端技术基线。

## 6. 状态与错误处理

原型需要覆盖以下状态：

- 纪要列表为空。
- 筛选后无结果。
- RAG 无引用命中。
- 录制程序未配置。
- 录制程序配置检查失败。
- 录制程序配置已保存但未校验。
- 录制向导未完成。
- 模拟录制中断。
- 草稿未保存。
- 草稿已保存但未发布。
- 发布后形成新版本。

所有模拟状态应明确表达“原型状态”或“草稿状态”，避免让操作员误以为已经产生真实录制、真实发布或真实审计记录。

## 7. 安全与架构边界

本轮只做前端原型，边界如下：

- 不新增后端 API。
- 不新增数据库表。
- 不新增 Skill。
- 不调用 Worker。
- 不调用 Python 录制程序。
- 不读取本机文件系统。
- 不启动本机进程。
- 不请求麦克风权限。
- 不连接真实模型或向量库。
- 不保存真实会议内容。
- 不引入多租户、外部客户接入或商业化能力。

后续真实接入时必须另行设计：

- 录制程序如何通过 M07 受限执行或受控本机代理接入。
- 客户端 Python 录制程序配置如何被校验、签名、审计和回滚。
- 本机受控桥接程序如何限制可执行路径、参数和输出目录。
- 纪要、转写、附件和向量索引的事实源与对象存储方案。
- RAG 查询契约、引用契约和评测集。
- 草稿发布、版本回滚、权限、审计和保留策略。
- 敏感内容脱敏、访问控制和导出限制。

## 8. 验收标准

前端原型验收：

- `/meeting-notes` 展示纪要库、基础筛选、纪要列表、开始录制入口和全局会议问答。
- 页面展示客户端录制程序配置状态，并能进入配置页。
- 录制程序配置页能模拟保存 Python 路径、脚本路径、工作目录和输出目录。
- 录制程序配置页明确展示配置归属为当前操作员和当前客户端 PC。
- 未完成录制程序配置时，录制向导展示明确的配置缺口。
- 筛选条件能基于本地模拟数据更新列表。
- 点击纪要能进入 `/meeting-notes/:id`。
- 详情页第一屏展示摘要、关键决策、行动项、参会人和版本状态。
- 详情页展示全文转写、版本记录和单场会议问答。
- 录制向导能完成配置检查、音频来源、会议元信息和总结模板选择的模拟流程。
- 校订页能保存草稿并模拟发布新版本。
- RAG 回答展示引用来源；无引用时展示明确空结果。
- 页面受 `ProtectedRoute` 保护。
- 测试覆盖路由、筛选、详情渲染、录制向导和草稿发布模拟状态。

浏览器验收至少覆盖当前桌面工作区视口。若移动端不作为本轮范围，应在验收记录中明确说明。

# ADR 0009：SQL 工作台开发环境受控 CRUD 与会话模型

- 状态：Accepted
- 日期：2026-06-13
- 负责人：平台与安全负责人
- 相关模块：M01、M02、M05、M07、M08、M09、M10、M11
- 相关任务：SQL 工作台目标态

## 背景

仅提供 SQL 校验和 DML 预检会迫使研发工程师切换到其他数据库客户端完成开发环境写操作，无法实现通过统一工作台和大模型分析减少缺陷、提升效率的产品目标。

## 决策

- SQL 工作台以主流数据库客户端交互为基线：对象浏览器、多 SQL 文件标签、传统编辑器、执行工具栏和底部结果区；AI 作为右侧横向折叠增强层提供错误分析与性能优化，收起后必须释放空间给编辑器和结果区。
- P1 保持只读执行边界，DML 左侧语义按钮只能触发预检。
- P2 开放开发环境的受控 `INSERT`、`UPDATE`、`DELETE` 执行；测试环境 DML 根据风险策略进入确认、审批或拒绝。
- 生产 SQL 连接在所有阶段均不可见、不可配置、不可调用。
- 每个 SQL 文件标签拥有独立上下文、分析报告和运行状态；多个只读查询可以并行执行和独立取消。
- 工作台会话不持有长期数据库连接。DML 使用单次短事务，成功自动提交，失败自动回滚。
- 默认不提供跨标签手动事务、长期事务、`Commit` 或 `Rollback`。
- 大模型只提供不可信分析建议；授权决策始终由服务端策略完成。

## 写执行门禁

P2 开发环境 DML 执行前必须满足：

1. SQL 可被可靠解析，并且属于允许的单条 DML 子集。
2. 目标环境、连接、Schema、表、字段和操作人均通过服务端策略。
3. 预检报告包含影响行数、脱敏样本或无法验证项。
4. SQL、参数哈希、连接、环境、策略版本和预检报告已绑定。
5. 持久化工作流、幂等键、审计事件、超时和失败恢复语义已建立。
6. Worker 使用受限写账号，并在执行前再次校验环境和语句类型。
7. 执行使用短事务，禁止向用户暴露未受控的长期事务句柄。

任一关键内容变化后，已有分析、确认和审批自动失效。

## 影响

- 需要为 SQL 会话、SQL 文件标签、DML 分析、执行决策和执行结果新增版本化强类型契约。
- 需要将 SQL 工作流接入 M05 持久化事实源，而不是依赖浏览器状态或内存结果。
- 需要按环境与风险拆分数据库凭据和最小权限账号。
- P2 开发写执行会触及 M02、M05、M07 和 M08 安全关键边界，必须经过安全评审。

## 明确不做

- 不开放生产 SQL 连接。
- 不执行任意多语句脚本。
- 不允许模型自动授权或自动执行 DML。
- 不在目标态第一阶段提供交互式长期事务。
- 不因工作台能力扩展新增部署服务或模块编号。

## 发布与回滚

P2 写执行能力必须使用独立功能开关按环境启用，默认关闭。回滚时关闭开发和测试 DML 执行能力，保留只读查询与预检；同时撤销 Worker 写凭据和未完成执行授权。

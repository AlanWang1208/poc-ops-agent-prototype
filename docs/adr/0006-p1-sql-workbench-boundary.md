# ADR 0006：P1 SQL 工作台边界与 Db2 for i 适配方案

- 状态：Accepted
- 日期：2026-06-13
- 负责人：平台与安全负责人
- 相关模块：M01、M02、M05、M07、M08、M09、M11
- 相关任务：P1 SQL 工作台

## 背景

研发工程师需要在公司内部平台中查询 AS/400 开发和测试环境，并在执行前检查
`INSERT`、`UPDATE`、`DELETE` 的语法与风险。P1 明确禁止生产写执行、任意脚本执行和绕过策略或审计的路径。

## 决策

- SQL 工作台是 M09 中的独立产品能力，但不新增部署服务或模块编号。
- P1 仅暴露 AS/400 开发和测试连接，生产连接不得进入连接目录、控制面 API 或 Worker 配置。
- 控制面使用 Apache Calcite 对单条 SQL 做保守 AST 分类。无法可靠解析或不在允许子集内的语句直接拒绝。
- P1 仅允许单条 `SELECT` 进入执行信封；`INSERT`、`UPDATE`、`DELETE` 仅生成静态预检报告。
- Worker 使用独立 Calcite 只读校验再次验证信封，避免绕过控制面后执行写语句。
- Db2 for i 使用 JTOpen JDBC 适配。控制面不持有数据库凭据，Worker 只接受凭据引用。
- P1 真实联调使用 Java KeyStore 保存凭据，由管理员启动 Worker 时人工解锁。P2 前必须替换为无人值守安全解锁，否则不得进入恢复与可用性验收。

## 依赖评审

### Apache Calcite 1.42.0

- 用途：SQL AST 解析和语句分类。
- 许可证：Apache License 2.0。
- 安全影响：依赖树较大；不得把 Calcite 的解析成功等同于 Db2 for i 运行成功，也不得把解析器作为唯一只读防线。
- 现有技术栈不足：JDK 与 Spring 不提供可用于安全分类 SQL 的结构化解析器；正则表达式不满足信任边界要求。

### JTOpen 21.0.6

- 用途：Worker 通过 JDBC 连接 Db2 for i。
- 许可证：IBM Public License 1.0。
- 安全影响：只允许在受限 Worker 中加载；数据库账号自身必须只读；连接串、用户名和凭据不得进入日志或事件。
- 现有技术栈不足：Java 不原生提供 IBM i JDBC 驱动，ODBC 会引入额外原生运行时和部署边界。

## 影响

- SQL 工作台请求使用独立版本化契约，不伪装成普通 Skill。
- 当前实现完成连接目录、AST 校验、DML 静态预检和 Worker 双重拒绝边界。
- 当前实现尚未完成真实 AS/400 查询结果分页、脱敏、短期结果存储、KeyStore 解锁和查询工作流持久化；这些能力完成前，真实执行器必须保持不可用并明确失败。
- SQL 工作台目标态的开发环境受控 CRUD、语义执行轨道和 Notebook 会话模型由 ADR 0007 定义；这些能力不改变 P1 的只读执行边界。

## 验证方式

- 自动化测试证明生产环境请求、DML 执行、多语句、DDL 和过期 Worker 请求被拒绝。
- 自动化测试证明允许的开发测试连接和单条 SELECT 可以通过校验。
- 真实 AS/400 联调必须单独记录 Db2 for i 方言、权限、超时、取消、结果限制和 Explain 行为证据。

## 发布与回滚

发布时默认只启用连接目录与校验 API，不配置真实 AS/400 凭据。回滚通过部署上一已验证版本完成，并撤销 Worker 中的连接与 KeyStore 配置。

# 前端对话框标准

## 适用范围

本标准适用于 `frontend/operator-console` 中需要临时覆盖当前页面的确认、详情、审计、审批和只读诊断信息展示。

M09 页面必须优先使用公共组件 `src/components/primitives/Dialog.jsx`，不得在业务页面内重复实现 `createPortal`、遮罩、滚动锁、Esc 关闭、标题语义或关闭按钮。

## 组件契约

公共对话框组件为 `Dialog`，核心属性如下：

| 属性 | 要求 |
|---|---|
| `open` | 控制是否渲染，对话框关闭时不保留 DOM |
| `onClose` | 统一关闭入口，Esc、遮罩点击和关闭按钮都调用该函数 |
| `title` | 必填，作为 `aria-labelledby` 的可访问名称 |
| `description` | 可选，作为 `aria-describedby` 的说明文案 |
| `eyebrow` | 可选，用于标识模块、阶段或上下文 |
| `size` | 可选，取值为 `compact`、`standard`、`wide`、`fullscreen` |
| `closeLabel` | 可选，关闭按钮的可访问名称 |

## 尺寸标准

| 尺寸 | 使用场景 |
|---|---|
| `compact` | 简短确认、提示、不可用原因 |
| `standard` | 默认详情、短表单、少量字段 |
| `wide` | 执行链、Skill、审计明细、两列信息区 |
| `fullscreen` | 大量日志、长表格、可视化流程或需要横向空间的工具 |

`/agent` 的对话执行、Skill 调用和执行链详情统一使用 `wide`。

## 行为标准

1. 对话框打开时必须锁定 `html` 和 `body` 滚动。
2. 对话框关闭时必须恢复打开前的滚动状态。
3. Esc、遮罩点击和关闭按钮必须走同一个 `onClose`。
4. 遮罩点击只能在点击遮罩本身时关闭，不得因为点击内容区冒泡误关闭。
5. 对话框内容区负责滚动，业务内容不应再创建无必要的嵌套滚动。
6. 不得在前端对话框中实现授权判断，只展示服务端返回的强类型状态和契约数据。

## 视觉标准

1. 标题区包含 eyebrow、标题、说明和图标关闭按钮。
2. 标题使用产品 UI 字号，不使用展示型大标题。
3. 背景采用克制的中性色和低饱和信息色，不使用大面积装饰性渐变。
4. 边框、阴影和圆角由公共组件统一控制。
5. 业务页面只负责内容排版，不覆盖公共对话框壳层样式。

## 动效标准

1. 动效只表达弹出和关闭状态，不做长时间编排动画。
2. 不动画布局属性。
3. 默认进入动效控制在 150 到 250 毫秒。
4. 必须支持 `prefers-reduced-motion`。

## 测试标准

公共组件测试必须覆盖：

1. `open=false` 时不渲染。
2. `open=true` 时渲染 `role="dialog"` 和 `aria-modal="true"`。
3. 标题、说明、eyebrow 和内容可见。
4. Esc、遮罩点击和关闭按钮都会触发 `onClose`。
5. 尺寸通过 `data-dialog-size` 暴露，便于页面测试确认标准使用情况。

业务页面测试必须覆盖：

1. 页面源码引入公共 `Dialog`。
2. 页面不得直接使用 `createPortal` 实现弹窗壳。
3. 页面不得保留私有遮罩和关闭按钮壳层样式。
